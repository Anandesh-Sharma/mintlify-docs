---
title: 'Create Invites'
openapi: 'POST /events/{event_id}/invites'
---

This endpoint allows you to create a batch of invites for a specific event. You can specify the onboarding flow type and notification settings.

## Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `event_id` | string | Yes | The unique identifier of the event |

## Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `onboarding_flow_type` | string | No | The type of onboarding flow to use. Defaults to "only_auth". Options include: "only_auth", "kyc", "ssn_verification", "peps_sanctions", "criminal_check" |
| `notify` | integer | No | Whether to send notifications. Defaults to 2 |

## Request Body

The request body should include an array of invite objects:

```json
{
  "invites": [
    {
      "first_name": "John",
      "last_name": "Doe",
      "email": "john.doe@example.com",
      "country_code": "1",
      "mobile_number": "5551234567",
      "authentication_method": "QR Code"
    },
    {
      "first_name": "Jane",
      "last_name": "Smith",
      "email": "jane.smith@example.com",
      "country_code": "1",
      "mobile_number": "5559876543",
      "authentication_method": "Biometric"
    }
  ]
}
```
