---
title: 'Get Attendees'
openapi: 'GET /attendees/{event_id}'
---

This endpoint retrieves a list of all people who have registered for a specific event. This is different from invites - these are confirmed attendees who have completed registration.

**What it does:**
- Provides a complete list of registered attendees for an event
- Includes detailed information about each person (name, email, ID, etc.)
- Shows when each person registered

**When to use it:**
- When preparing for an event to know who's coming
- For generating name badges or check-in lists
- When you need to contact all registered attendees
- For reporting on event attendance

## Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `event_id` | string | Yes | The unique identifier of the event |

## Response

The API returns an array of attendee objects with their complete details:

```json
[
  {
    "id": "attendee-123",
    "first_name": "John",
    "last_name": "Doe",
    "email": "john.doe@example.com",
    "ticket_number": "TICKET123",
    "user_id": "user-456",
    "img_url": "https://example.com/profile-photos/john-doe.jpg",
    "registration_time": "2023-02-25T14:30:00Z"
  },
  {
    "id": "attendee-124",
    "first_name": "Jane",
    "last_name": "Smith",
    "email": "jane.smith@example.com",
    "ticket_number": "TICKET124",
    "user_id": "user-457",
    "img_url": "https://example.com/profile-photos/jane-smith.jpg",
    "registration_time": "2023-02-26T09:15:00Z"
  }
]
```

**Important fields:**
- `ticket_number`: A unique identifier for each attendee's ticket
- `img_url`: Profile photo that can be used for visual identification
- `registration_time`: When the person completed their registration

Use this information to prepare for your event, create check-in sheets, or generate attendance reports.
